匿名 Agent 系统示意图

这里只展示通用结构：姓名、私人项目、兴趣偏好、账号凭证、本地路径和敏感领域都已替换成中性示例。

中控台 / Control Center

负责 registry、routing、shared rules、status snapshot、operation log，以及跨 Agent 协调。

研究与知识 Agent

负责阅读、笔记、来源审查、方法整理、写作和可复用知识记忆。

文献与资料综述来源筛选、核心论点、方法、引用
Source Cards稳定笔记、来源记录、可复用摘要
方法笔记分析套路、示例、限制条件
写作支持提纲、修改、审稿式检查

工作与公开材料 Agent

负责公开文档、机会追踪、匹配判断、材料版本和后续跟进。

机会入口官方来源、要求、截止日期
个人事实库已确认事实、版本、证据链接
匹配分析匹配点、缺口、风险、下一步问题
状态追踪进度、日期、待审队列
沟通草稿消息、交接、人类确认

生活事务 Agent

负责表格、预约、家庭事务、清单和周期性 follow-up。

文件清单表格、证件、证明、缺失项
预约规划日期、地点、准备事项
家庭事务笔记共享上下文，但不乱扩散隐私
跟进队列电话、邮件草稿、等待状态
私密事务子 Agent为敏感任务隔离上下文

学习 Agent

负责课程、练习记录、错因复盘、学习计划和进度记忆。

课程笔记概念、例子、阅读材料
练习复盘尝试记录、用时、趋势
错因日志错误、原因、复习日期
学习计划场次、优先级、复习节奏
陪练模式解释、提问、调整难度

创作与写作 Agent

负责草稿、语气笔记、编辑记忆、发布准备和修改习惯。

草稿台想法、提纲、版本、片段
风格指南语气、禁用表达、好例子
修改队列逐句修改、结构、事实检查
发布清单格式、署名、最终审阅

自动化实验室

负责脚本、API、浏览器任务、验证器、dashboard 和发布前卫生检查。

脚本运行器先 dry-run，再审阅执行
API 连接器密钥放 env，输出进 workspace
浏览器流程采集、截图、来源捕捉
健康检查断链、陈旧任务、被追踪的 secrets
发布前检查license、测试、私密数据扫描

知识蒸馏器

把笔记、prompt、模板和外部示例变成可复用的系统零件。

Prompt Inbox原始 prompt 先按不可信数据处理
Knowledge Cards稳定事实、规则、例子
Skill Fusion谨慎合并重复流程
Template Review范围和安全检查后再采用

项目工作区

临时或高上下文项目，不必永久挤进主 Agent 的记忆里。

Project Alpha限时文件、task.yaml、outputs
协作沙盒共享上下文和清楚的 handoff notes
归档通道已完成任务、可复用经验、旧草稿

每个主 Agent 通常包含

入口AGENTS.md / CLAUDE.md
控制IDENTITY.md + RULES.md
状态STATUS.md + task.yaml + MEMORY.md
执行skills/ + scripts/ + subagents/
沉淀knowledge/ + raw_data/ + outputs/

读图方式

  • Root 负责路由和系统地图，不是什么活都亲自干。
  • Main agents 把长期上下文分开，避免 memory 变成一锅粥。
  • Subagents 用来隔离长期、敏感或高上下文项目。
  • 默认只有 outputs/ 里经过审阅的文件适合对外分享。

知识积累体系

1. 输入raw_data / inbox / source links / chats / screenshots
2. 当前工作workspace/、活跃草稿、trackers、中间笔记
3. 稳定知识knowledge/ 保存事实、来源规则和可复用上下文
4. 长期笔记vault/ 或 archive/ 保存已审阅且值得留下的材料
5. 复盘回路source cards、method notes、output checks、经验沉淀
6. 技能固化重复流程经过审阅后升级为 skills/

Inter / Intra-Agent Skill 调用

Local Skills任务进入某个 Agent 后，由 RULES 和 task.yaml 选择最小可复用流程。
Script / API Skills自动化通过已审阅脚本、dry-run、环境变量和输出检查执行。
Handoffs跨领域任务回传摘要、已验证产物和路径，而不是到处倾倒原始上下文。
